\# SQL Project: Parks and Recreation Database (Beginner)



Este repositório contém o ecossistema de aprendizado e consolidação em SQL (MySQL), utilizando o clássico banco de dados fictício da série \*Parks and Recreation\*. O projeto documenta a minha evolução prática, partindo da fundação dos bancos de dados relacionais até técnicas avançadas de manipulação de dados.



\---



\## Estrutura do Banco de Dados



O banco é composto por 3 tabelas principais, desenhadas estrategicamente para praticar cruzamento de dados, valores nulos (`NULL`) e inconsistências propositais do mundo real:



\*   \*\*`employee\_demographics`\*\*: Dados pessoais dos funcionários (ID, nome, idade, gênero e data de nascimento).

\*   \*\*`employee\_salary`\*\*: Dados financeiros e profissionais (ID, cargo, salário e ID do departamento).

\*   \*\*`parks\_departments`\*\*: Tabela de referência dos departamentos disponíveis na cidade de Pawnee.

\## Habilidades Desenvolvidas (Grade Curricular)



\---



\### Nível Beginner (Concluído)

\*   \*\*DDL \& DML\*\*: Criação de bancos de dados, modelagem de tabelas (`CREATE TABLE`), definição de Chaves Primárias (`PRIMARY KEY`) e inserção de registros (`INSERT INTO`).

\*   \*\*Querying Básica\*\*: Domínio completo do comando `SELECT` e boas práticas de formatação e quebra de linhas para legibilidade.

\*   \*\*Filtragem de Dados\*\*: Uso cirúrgico da cláusula `WHERE` com operadores de comparação (`>`, `>=`, `!=`), manipulação de formatos de data padrão (`AAAA-MM-DD`) e filtros dinâmicos de texto com `LIKE` e caracteres coringa (`%` e `\_`).

\*   \*\*Agregações e Sumarização\*\*: Agrupamento de registros idênticos usando `GROUP BY` combinado com funções matemáticas fundamentais (`AVG`, `SUM`, `MIN`, `MAX`, `COUNT`).

\*   \*\*Ordem de Execução (SQL Pipeline)\*\*: Entendimento prático de por que o `WHERE` falha com funções agregadas e como corrigir isso aplicando o `HAVING`.
